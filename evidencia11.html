4.2.- Práctica: Demostración de Inyección SQL

Fecha: Febrero / 2026

Descripción

Crear una pequeña aplicación web en PHP con un login vulnerable a inyección SQL para probar ataques usando comillas y comentarios (como admin' --) para burlar la autenticación sin saber la contraseña.

Resultados Obtenidos

Se logró un inicio de sesión exitoso simulando el ataque. Luego se corrigió el archivo login.php implementando Consultas Preparadas (Prepared Statements con bind_param), logrando sanitizar los datos y bloquear la vulnerabilidad por completo.

Reflexión Personal

"Hacer este código me sirvió para entender que jamás hay que concatenar variables directo en los strings de las consultas SQL. Las consultas preparadas salvan aplicaciones enteras."
